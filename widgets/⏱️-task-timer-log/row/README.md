# ⏱️ Task Timer（Log）

Notion に埋め込み可能なタスクタイマーです。
Stop 操作ごとにログを追加し、Clear するまで記録を積み上げる仕様になっています。

日をまたぐ作業や、実務ベースのタスク管理を想定して設計しています。

# 特徴

Stop でログを追加

ログは Clear するまで保持

Total は 下に表示されているログ全体の合計

日付をまたいでもログは継続

データは ブラウザの localStorage に保存

Notion iframe 埋め込み最適化

# 基本仕様（共通）
タイマー操作
操作	内容
Start	計測開始
Pause	一時停止（再開可）
Stop	計測を終了しログに追加
Reset	現在の計測を破棄（ログには残らない）
Clear	すべてのログを削除

# ログの考え方

1回の Stop = 1ログ

ログ時刻は「スタート時刻」

日付単位では分けない

Clear するまで累積される

ログのイメージ：

2026-01-15 13:19  Task A   00:12:30
2026-01-16 00:05  Task B   00:03:10

# Copy log（コピー内容）

Copy log ボタンでは、以下の形式でコピーされます。

Total: 01:23:40
2026-01-15 13:19 | 00:12:30 | Task A
2026-01-16 00:05 | 00:03:10 | Task B


日報

作業ログ

業務メモ
などにそのまま貼り付けて使えます。

# レイアウト別の違い

このウィジェットは レイアウト（形）違い を用意しています。
機能・挙動は同一です。

# 🔹 Row（横長レイアウト）

特徴

横長・2カラム構成

Notion ダッシュボード向き

作業中にログと合計を同時に確認しやすい

想定用途

デスクトップ中心

ダッシュボード常設

作業ログを常に見たい場合

ディレクトリ例：

task-timer-log/row/en/green-black/

# 🔹 Column（縦長レイアウト）

特徴

縦長・1カラム構成

スマホや狭い Notion ページ向き

シンプルに時間計測したい場合

想定用途

モバイル

縦スクロール中心のページ

軽量なタイムトラッキング

ディレクトリ例：

task-timer-log/column/en/green-black/

# データ保存について

保存先：ブラウザの localStorage

同じブラウザ・同じ端末でのみ保持されます

ブラウザデータ削除でログも消えます

※ クラウド同期は行いません。

# 設計思想

無理に管理しない
作業の流れを止めない
必要な情報だけ、静かに残す

実務の中で自然に使えることを重視しています。

# カスタマイズについて

配色変更のみ行う場合は、
CSS のカラー定義（:root）だけを差し替えてください。

HTML 構造・JavaScript ロジックの変更は推奨していません。
